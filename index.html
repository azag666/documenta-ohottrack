Guia de Integração e IA

Documentação técnica oficial para criação de checkouts web e Mini Apps utilizando a API da HotTrack. Projetado para desenvolvedores e modelos de inteligência artificial.

Autenticação

Todas as requisições para a API da HotTrack (exceto rotas públicas específicas) devem ser autenticadas enviando a sua chave secreta no header da requisição.

Headers
x-api-key: sua_chave_secreta_aqui
Content-Type: application/json

1. Resolve Click (Mini Apps)

Utilizada no início do funil para capturar e registrar a sessão do usuário, retornando o identificador de rastreamento (click_id).

POST https://hot-track.com/api/miniapp/resolve-click
Parâmetro (Body)TipoDescrição
initDataOBRIGATÓRIOstringDados de inicialização vindos do ambiente do Telegram.
checkoutIdOBRIGATÓRIOstringO ID do seu produto/checkout configurado na HotTrack.
Response (200 OK)
{
    "click_id": "clk_01HFWJ4..."
}

2. Geração de PIX

Gera a ordem de pagamento no gateway e retorna o código Copia e Cola para o cliente.

POST https://hot-track.com/api/pix/generate
Parâmetro (Body)TipoDescrição
click_idOBRIGATÓRIOstringID capturado no passo Resolve Click.
value_centsOBRIGATÓRIOintegerValor exato da cobrança em centavos (Ex: R$ 97,50 = 9750).
Response (200 OK)
{
    "transaction_id": "tx_01HFW...",
    "qr_code_text": "00020101021226870014br.gov.bcb.pix.gui..."
}

3. Status do Pagamento

Rota para checagem (polling) do status da transação. Deve ser chamada a cada X segundos após a exibição do PIX.

GET https://hot-track.com/api/pix/status/{id}
Response (200 OK)
{
    "status": "paid" // Pode retornar "pending", "paid", ou "expired"
}

Prompt Mestre para IA (Claude / Windsurf / Cursor)

Acelere seu desenvolvimento. O prompt abaixo foi desenhado com engenharia de contexto para garantir que a IA gere um código que respeite as regras de negócio da HotTrack, além de aplicar um design premium instantaneamente.

🤖 Prompt de Geração de Checkout

Substitua os valores entre colchetes [ ] antes de enviar para a IA.

Atue como um Engenheiro Front-end Sênior especialista em conversão e UI/UX. Preciso que você desenvolva um checkout web (Single Page Application em HTML, CSS embutido e Vanilla JS) integrado à API da plataforma HotTrack para vender o produto: [NOME DO PRODUTO]. DIRETRIZES DE DESIGN (CRÍTICO): O design deve ser rigorosamente minimalista e sofisticado, inspirado no ecossistema FinTech (como Stripe e Linear). - Utilize Dark Mode como padrão (fundo quase preto, superfícies cinza muito escuro). - Tipografia limpa (Inter ou system-ui). - Sem cores neon, sem sombras exageradas, sem elementos poluídos. O layout deve transmitir extrema confiança, tecnologia e segurança. REGRAS DE NEGÓCIO E FLUXO TÉCNICO (CRÍTICO): Você NÃO deve utilizar pagamentos nativos do Telegram em nenhuma hipótese. O fluxo de pagamento será exclusivamente via PIX Copia e Cola, executado na própria interface web, seguindo estes passos sequenciais: 1. CAPTURA (Resolve Click): Ao inicializar, faça uma requisição POST para 'https://hot-track.com/api/miniapp/resolve-click'. Envie no JSON body: { "initData": "[SUA_LOGICA_DE_INIT_DATA_AQUI]", "checkoutId": "[SEU_CHECKOUT_ID_AQUI]" }. Extraia o 'click_id' da resposta. 2. GERAÇÃO DA COBRANÇA: Com o 'click_id' em mãos, faça um POST para 'https://hot-track.com/api/pix/generate'. Headers obrigatórios: 'x-api-key: [SUA_API_KEY_AQUI]' e 'Content-Type: application/json'. JSON Body: { "click_id": "valor_capturado", "value_cents": [VALOR_EM_CENTAVOS_AQUI] }. Extraia o 'transaction_id' e o 'qr_code_text'. 3. INTERFACE DE PAGAMENTO: Exiba o código PIX ('qr_code_text') de forma clara com um botão proeminente "Copiar Código PIX". Ao clicar, o código deve ir para a área de transferência do usuário com um feedback visual elegante de sucesso. 4. POLLING DE STATUS: Imediatamente após renderizar o PIX, inicie um setInterval de 4 em 4 segundos fazendo um GET para 'https://hot-track.com/api/pix/status/{transaction_id}' passando o header 'x-api-key'. Se a API retornar { "status": "paid" }, interrompa o polling (clearInterval) e alterne a tela suavemente para uma mensagem de "Pagamento Confirmado" com um botão de acesso ao produto. Gere o código completo, blindado contra erros (com try/catch nas requisições) e pronto para ser executado no navegador.
