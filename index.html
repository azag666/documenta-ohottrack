Guia de Integração: ManyChat

Siga este guia visual para configurar a automação de pagamentos PIX. As telas abaixo são uma réplica exata do ManyChat para facilitar a sua implementação.

0

Preparação: Criando os Campos

Antes de tudo, precisamos criar os "campos" no ManyChat para guardar as informações. Faremos isso uma única vez.

1. Bot Fields (Suas Chaves Secretas)

Vá em Configurações (Settings) → Campos (Fields). Crie os 2 campos abaixo para guardar suas chaves da HotTrack e do Telegram.
É crucial que os nomes sejam exatamente estes:

  • keyhottrack (Tipo: Text) - Cole aqui sua chave da API HotTrack.
  • tokenbot (Tipo: Text) - Cole aqui o token do seu bot do Telegram.

2. User Fields (Dados dos Clientes)

Na mesma tela, na aba User Fields, crie estes 5 campos para salvar as informações de cada cliente durante a venda.

  • cidade (Tipo: Text)
  • click_id (Tipo: Text)
  • id (Tipo: Text)
  • qr_code (Tipo: Text)
  • status (Tipo: Text)
1

Gatilho (Trigger)

O fluxo começará quando um novo usuário entrar em contato pelo Telegram. Configure o gatilho (Trigger) como "New contact" e adicione a condição "Opted-in for Telegram is true".

2

Coletar Click ID

Esta ação salva o código de rastreio do seu lead. Adicione uma ação "Set User Field" e configure-a para salvar a variável {{Last Text Input}} dentro do campo de usuário click_id que criamos.

3

API: Gerar Cobrança PIX

Crie a cobrança PIX com o valor do produto. Lembre-se que o valor deve ser em centavos (ex: R$ 19,99 = 1999).

External Request
POST

Precisamos de duas chaves: uma para autenticação e outra para informar o formato do conteúdo.

KEY
VALUE

Enviamos o ID do lead e o valor da compra (em centavos).

{
    "click_id": "{{click_id}}",
    "value_cents": 1999
}

Salvamos o ID da transação e o código PIX nos campos de usuário.

5

API: Enviar PIX ao Cliente

Agora, usamos a API oficial do Telegram para enviar a cobrança ao cliente com um botão "copia e cola".

External Request
POST

Este corpo (Body) envia a mensagem formatada para o usuário no Telegram.

{
    "chat_id": "{{telegram_user_id}}",
    "text": "Seu código PIX Copia e Cola está pronto:\n\n{{qr_code}}",
    "parse_mode": "HTML",
    "reply_markup": {
        "inline_keyboard": [[
            { "text": "📲 Copiar Código PIX", "copy_text": "{{qr_code}}" }
        ]]
    }
}

7

Dividindo o Fluxo com Condição

Com o status salvo, use um bloco de "Condition" para direcionar o usuário para o caminho correto.

Crie duas condições para verificar o campo {{status}}:

  • ➡️ SE o campo {{status}} for paid → envie o usuário para o fluxo de "Pagamento Aprovado".
  • ➡️ SE o campo {{status}} for pending → envie o usuário para o fluxo de "Cobrança/Lembrete".
